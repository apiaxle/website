The Repl

You probably want this first. The repl allows you to configure ApiAxle from the command line. Setting up an API and API keys ready for the proxy to work with is easy. It fires up an instance of ApiAxle's own HTTP API in the background and uses that to modify aspects of the system. Anything you can do in the REPL you can do programmatically with the API too. 
$ sudo npm install apiaxle-repl
$ apiaxle

axle> key "05050c14643dc" create
axle> api "acme" create endPoint="localhost:81"
axle> api "acme" linkkey "05050c14643dc"

The Proxy

The main part of the system. This goes between the Internet and your API giving you things you don't want to have to write yourself like authentication, throttling, caching and statistics collection. It's fast, secure and easy to setup. 
$ sudo npm install apiaxle-proxy
$ apiaxle-proxy

The API

ApiAxle's own HTTP API which gives you full control over your APIs and the API keys used to access them. You can view statistics about individual APIs and API keys from week long granularities right down to near-real-time hits at a second granularity. 
$ sudo npm install apiaxle-api
$ apiaxle-api
$ curl localhost:3000/v1/apis
